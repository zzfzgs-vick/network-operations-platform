## DAG 检查规则

- 每张 Ticket 的全部依赖编号小于自身编号。
- Schema/合同 Ticket 位于使用方之前。
- UI Ticket 位于对应 API/领域合同之后。
- 容量数据工具 T058 位于目标容量 T059 和压力验收 T060 之前。
- 新增或修改依赖边后，必须重新执行文档自检并确认无循环。
