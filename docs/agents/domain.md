# Domain docs

This repository uses a single shared domain context.

## Read before working

- Read `docs/CONTEXT.md` for project scope and shared terminology.
- Read relevant documents under `docs/domain/`.
- Read applicable ADRs under `docs/architecture/adr/`.
- Read formal specifications under `docs/specs/` when a task references one.

If a document does not exist, continue without inventing its contents.

## Vocabulary

Use the canonical terms defined in `docs/CONTEXT.md` and `docs/domain/`. Note genuine terminology gaps instead of silently introducing synonyms.

## ADR conflicts

Surface any conflict with an accepted ADR explicitly. Do not silently override an architectural decision.
